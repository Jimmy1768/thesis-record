# Audit

Protected actions must create audit events.

At minimum, audit trails should cover source registration, intake manifest
changes, private artifact handling, quality review creation, evidence snapshot
generation, claim review decisions, export artifact generation, and publication
or update workflows.

Raw external research returns are audit/process artifacts, not evidence. They
should remain visible under thesis-local operator records even when rejected.
Their interpreted intake notes should record source leads, methodology
candidates, open questions, counterarguments, rejected items, and immediate
allowed uses.

See `docs/source-verification/raw_return_buffer_policy.md`.
