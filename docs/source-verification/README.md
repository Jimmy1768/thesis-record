# Source Verification

ThesisRecord treats source verification as a first-class workflow.

Source records should preserve source-native context, access paths, limitations,
quality reviews, and failure records. They must not promote thesis claims until
the relevant claim-review gates are satisfied.

External research returns must pass through the raw-return buffer before they
can influence accepted thesis material. Raw returns are process artifacts;
intake notes classify them; only independently inspected sources can become
source notes, evidence, methodology, claims, or prose.

Public source manifests belong under the thesis whose evidence they support.
Shared process templates belong under root `templates/`.
